## 내가 가장 잘 처리하는 요청 방식

### 1. 슬라이드 번호로 지정

예시:

```text
13페이지 열어서 메인 제목을 "Swimming Start"에서 "Swimming Start."로 바꿔줘.
```

```text
2페이지 프로필 슬라이드에서 자격증 문구를 더 자연스럽게 다듬어줘.
```

### 2. 프레임 이름으로 지정

예시:

```text
NOOS — 4-6 TROUBLESHOOTING 페이지에서 첫 번째 이슈 제목을 더 짧게 바꿔줘.
```

```text
마무리 — CLOSING 페이지의 마지막 문구를 조금 더 강하게 수정해줘.
```

### 3. Figma URL 또는 node-id로 지정

예시:

```text
node-id `219:225` 프레임에서 하단 페이지 번호 스타일 맞춰줘.
```

```text
이 URL의 프레임 열어서 제목 정렬 수정해줘:
https://www.figma.com/design/axMbrfOMtVQxetuY0pXMdq/...?...node-id=219-225
```

## 요청할 때 같이 주면 좋은 정보

아래 항목이 많을수록 내가 바로 수정하기 쉽다.

1. 어디를 수정할지
   예: `13페이지`, `프로필`, `node-id 35:2`
2. 무엇을 바꿀지
   예: `제목`, `본문`, `사진`, `색상`, `간격`, `정렬`
3. 어떻게 바꿀지
   예: `문구 교체`, `더 짧게`, `더 고급스럽게`, `왼쪽 정렬`, `폰트 크기 키우기`
4. 기준 문구 또는 대상 문구
   예: `"진행중"을 "In Progress"로 변경`
5. 유지해야 할 조건
   예: `전체 톤은 유지`, `영문 소문자 유지`, `현재 배치 최대한 유지`

## 가장 좋은 요청 템플릿

```text
[슬라이드 번호/프레임 이름/node-id]에서
[수정할 요소]를
[어떻게 바꿀지] 수정해줘.
현재 문구는 "[기존 내용]"이고,
바꿀 문구는 "[새 내용]"이야.
전체 톤은 [유지/변경]해줘.
```

예시:

```text
10페이지 NOOS — 4-6 TROUBLESHOOTING에서
첫 번째 문제 상황 문구를
조금 더 간결하게 수정해줘.
현재 문구는 "토큰 만료 시 자동 로그아웃이 일어나 세션 단절."이고,
바꿀 문구는 "토큰 만료 시 세션이 즉시 끊겨 사용 흐름이 깨졌다."야.
전체 톤은 유지해줘.
```

## 내가 바로 처리 가능한 작업

- 슬라이드 찾기
- 특정 프레임 열기
- 텍스트 수정
- 문장 다듬기
- 제목/소제목 교체
- 정렬, 간격, 크기 같은 레이아웃 수정
- 색상, 강조 톤, 카드 구조 같은 시각 수정
- 페이지 복제 후 변형안 만들기

## 애매하게 말하면 확인이 느려지는 경우

아래처럼 말하면 대상이 여러 개일 수 있다.

```text
그 중간 페이지 좀 고쳐줘.
```

```text
NOOS 쪽 문구 바꿔줘.
```

이 경우는 아래처럼 바꾸면 바로 처리할 수 있다.

```text
7페이지 NOOS — 4-3 ARCHITECTURE에서 왼쪽 설명 문구 바꿔줘.
```

```text
18페이지 Swimming Start — 5-6 TROUBLESHOOTING에서 두 번째 이슈 제목 바꿔줘.
```

## 추천 방식

앞으로는 아래 둘 중 하나로 말하면 된다.

1. `몇 페이지` 기준
   예: `13페이지 제목 바꿔줘`
2. `프레임 이름` 기준
   예: `프로필 페이지 자격증 섹션 수정해줘`

가장 정확한 방식은 아래다.

```text
18페이지 (`219:225`)에서 두 번째 이슈 제목을 더 명확하게 바꿔줘.
```
